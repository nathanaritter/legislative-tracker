# Legislative Tracker

Internal Milestone tool for tracking CRE-relevant legislation across target markets (CO, TX, FL, UT, VA, MD, NV, TN, GA, AZ, ID). Dash app over a parquet-based bill catalog produced by `etl-base`.

## Quick start

```bash
pip install -r requirements.txt
cp .env.example .env
python app.py
```

Opens on http://localhost:8050. Reads parquet from `../etl-base/parquet/market/legislation/` when `USE_AZURE=false`, or from Azure Blob `stdp58902/etl/` when `USE_AZURE=true`.

## Architecture

See `CLAUDE.md` for the full design. Summary:

- **Data**: lives in `../etl-base/processed/market/legislation/` as `bills.csv` (reference table), `bill_events.csv` (long-format event log), and `metrics.csv` (aggregates for analytics-workbench). Generated by scripts in `etl-base/etl/legislation/` that wrap connectors from this repo's `ingest/`.
- **Ingest connectors** (`ingest/legiscan/`, `ingest/legistar/`): write raw API responses directly to Azure Blob. Nothing per-bill lands on OneDrive.
- **AI enrichment**: runs out-of-band as a scheduled Claude Code trigger. Summaries and risk scores are merged back into `bills.csv`.
- **UI**: timeline (Gantt), click → detail modal with summary, risk score, sponsors, full bill text link.
